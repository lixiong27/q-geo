**关联热词交互说明：**

1. 用户在搜索框输入关键词
2. 系统进行模糊搜索（防抖 300ms）
3. 下拉显示匹配的热词列表，每项显示：热词内容 + 类型标签
4. 用户点击选择后，搜索框隐藏，显示已选热词卡片
5. 卡片显示：热词内容、类型标签、热词ID
6. 用户可点击 [×] 清除已选热词，重新搜索

## 五、交互说明

### 5.1 热词管理页面

1. **类型筛选**：点击类型标签筛选，支持与来源筛选组合
2. **新增热词**：弹出表单，可选择热词类型
3. **编辑热词**：弹出表单，可修改热词类型

### 5.2 热词分析页面

1. **新建任务**：填写任务名称、类型、模型、数量后提交
2. **任务执行**：后台异步执行，状态实时更新
3. **查看结果**：已完成任务可查看分析出的热词列表
4. **导入选中**：勾选热词后导入到热词管理列表

## 六、API 调用流程

### 6.1 热词管理

```
页面加载 → getHotWordTypes() 获取类型配置
        → getHotWordList() 获取热词列表

筛选切换 → getHotWordList({ sourceType, type }) 刷新列表

新增热词 → addHotWord({ word, tags, type }) 提交
```

### 6.2 热词分析

```
页面加载 → getTaskList({ type: 'analysis' }) 获取任务列表
        → getHotWordTypes() 获取类型配置
        → getAnalysisModels() 获取分析模型列表（从 QConfig 动态获取）

新建任务 → 用户输入关键词搜索热词
        → getHotWordList({ keyword }) 模糊搜索已有热词
        → 用户选择热词，获取 hotwordId
        → createAnalysisTask({ hotwordId, name, type, model, count })
        → 任务开始执行

查看结果 → getTaskDetail(taskId)
        → 展示结果列表

导入选中 → importAnalysisResults(taskId, selectedWords)
```
